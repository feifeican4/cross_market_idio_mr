# 跨市场特质均值回归策略：中文理解说明

这份文档不是代码说明书，而是帮助你真正理解这道量化策略题。你可以把它当成面试前的讲稿，也可以一边看它一边读代码。

---

## 1. 这道题到底想考什么

这道题不是让你预测 BTC、MSTR、NVDA 明天涨还是跌。

它真正想考的是：

1. 你能不能把一个资产的涨跌拆成“市场带来的部分”和“资产自己的部分”。
2. 你能不能只交易“资产自己的异常偏离”，而不是暴露在大盘方向上。
3. 你能不能用严谨回测证明这个想法在扣掉成本后是否还有效。
4. 你能不能说清楚这个策略什么时候会失效。

所以这题的关键词不是“预测”，而是：

```text
因子剥离 + 残差交易 + 均值回归 + 对冲 + 成本后回测
```

---

## 2. 用一句话理解策略

找到那些相对自己因子模型涨多了或跌多了的资产，反向交易它们，同时用因子腿对冲市场风险，只赚残差回归的钱。

更直白一点：

```text
不是赌 MSTR 会跌，
而是赌 MSTR 相对 BTC 和 SPY 的过度上涨会收敛。
```

如果你只做空 MSTR，那你承担的是 MSTR、BTC、美股市场一起波动的风险。  
如果你做空 MSTR，同时按 beta 买入 BTC 和 SPY，你主要承担的是 MSTR 自己偏离模型的风险。

---

## 3. 什么是“特质”

每个资产的收益可以拆成两部分：

```text
资产收益 = 系统性收益 + 特质收益
```

系统性收益，是被大市场或大因子解释的部分，比如：

- BTC 涨，MSTR、COIN、MARA 也容易涨。
- QQQ 涨，NVDA、AMD、META 也容易涨。
- ETH 涨，AAVE、LINK 等以太坊生态资产也容易涨。

特质收益，是因子解释不了的部分，比如：

- MSTR 因为某个市场情绪被炒太高。
- COIN 因为加密交易量预期突然被重新定价。
- AAVE 因为 DeFi 情绪突然变化。

代码里，特质收益就是 residual：

```text
residual = actual_return - predicted_return
```

## 4. 为什么要建因子模型

假设今天：

```text
BTC 涨了 5%
MSTR 涨了 12%
```

如果历史上 MSTR 对 BTC 的 beta 大约是 2，那么 BTC 能解释的 MSTR 涨幅是：

```text
2 * 5% = 10%
```

那 MSTR 多出来的部分就是：

```text
12% - 10% = 2%
```

这个 2% 才是 MSTR 自己的异常偏离。

所以我们建模型：

```text
MSTR_return = alpha + beta_1 * BTC_return + beta_2 * SPY_return + residual
```

模型不是为了预测未来价格，而是为了把系统性风险剥离掉。

## 5. 为什么用滚动回归

不能用 2023 到 2026 的全样本一次性回归 beta，因为这会偷看未来。

正确做法是：

```text
在 t 日，只用 t 日之前的数据估计 beta。
然后用这个 beta 解释 t 日的资产收益。
```

项目里默认用 90 日滚动窗口：

```text
过去 90 个交易日估计 beta
今天计算预测收益
今天实际收益 - 预测收益 = 今天残差
```

这样做有两个好处：

1. 避免前视偏差。
2. 允许 beta 随市场状态变化。

对应代码：

```text
src/cross_market_mr/factor_model.py
fit_rolling_factor_model()
```

## 6. R² 是什么，为什么要报告

R² 衡量因子模型能解释资产波动的比例。

例如：

```text
R² = 0.60
```

意思是资产 60% 的波动可以被因子解释。

如果 R² 太低，说明模型没有很好剥离系统性风险，残差可能不是“特质收益”，而是漏掉了重要因子。

作业要求报告每个资产的 R²，因为它想看：

```text
你选的因子到底有没有解释力？
```

对应输出：

```text
reports/demo/diagnostics.csv
reports/live/diagnostics.csv
```

## 7. ADF 检验是什么，为什么要做

均值回归的前提是残差不能像随机游走一样一直漂。

ADF 检验就是检查一个时间序列是否更接近平稳序列。

你可以这样理解：

```text
如果残差是平稳的，它偏离均值后更可能回来。
如果残差是随机游走，它偏离后可能继续偏离。
```

通常：

```text
ADF p-value < 0.05
```

说明残差平稳性更强，对均值回归策略更友好。

但是要注意：

```text
ADF 通过不等于一定赚钱。
ADF 不通过也不等于一定不能交易。
```

它只是一个统计诊断。

## 8. z-score 信号怎么理解

残差本身有大有小，不同资产不能直接比较。

所以我们把残差标准化：

```text
z = (当前残差 - 过去残差均值) / 过去残差标准差
```

如果：

```text
z = +2.5
```

说明当前残差比历史均值高 2.5 个标准差，属于比较极端的高估。

策略规则：

```text
z > +2.0  做空资产，做多因子
z < -2.0  做多资产，做空因子
|z| < 0.5 平仓
```

为什么进场是 2.0，出场是 0.5？

因为如果一低于 2.0 就平仓，会频繁交易，手续费会很高。  
出场阈值设成 0.5，是等偏离明显收敛后再走。

```text

```

## 9. 对冲组合怎么构建

假设模型估计出来：

```text
MSTR = 2.0 * BTC + 0.3 * SPY + residual
```

如果信号认为 MSTR 残差太高，要做空 MSTR：

```text
short 1 美元 MSTR
long  2 美元 BTC
long  0.3 美元 SPY
```

如果信号认为 MSTR 残差太低，要做多 MSTR：

```text
long  1 美元 MSTR
short 2 美元 BTC
short 0.3 美元 SPY
```

统一公式：

```text
factor_weight = -asset_weight * beta
```

## 10. 为什么要做波动率缩放

不同资产的残差波动不同。

比如：

- NVDA 残差日波动可能是 2%。
- DOGE 残差日波动可能是 6%。

如果两个都放 3% 仓位，DOGE 的风险明显更大。

所以仓位要按残差波动率缩放：

```text
残差波动越大，仓位越小。
残差波动越小，仓位越大。
```

项目中使用：

```text
asset_weight = signal * capital_per_signal * target_pair_vol / residual_vol
```

然后再加上单一资产 3% 上限。

## 11. 风险约束为什么重要

作业要求：

```text
单一资产最大仓位 <= 3%
同一板块最大暴露 <= 15%
总杠杆 <= 3x
净因子暴露 <= 5%
```

这些约束不是装饰，而是为了防止策略变成方向赌博。

如果没有这些约束，可能出现：

- 同时大量做空矿企，等于押注 BTC 下跌。
- 同时大量做多科技股，等于押注 QQQ 上涨。
- 残差信号集中在一个板块，导致组合不分散。

对应代码：


## 12. 回测为什么要 shift 一天

这是最重要的前视偏差问题。

你用今天收盘价计算今天收益，得到今天残差和 z-score。  
这个信号是在今天收盘后才知道的。

所以你不能用今天信号赚今天收益。

正确逻辑：

```text
t 日收盘：计算信号
t+1 日：按信号持仓，赚 t+1 日收益
```

代码中：

```python
holdings = aligned_targets.shift(1).fillna(0.0)
```

对应文件：

```text
src/cross_market_mr/backtest.py
```

面试时你一定要主动讲这一点。

## 14. 敏感性分析怎么理解

如果一个策略只有在某一个参数下好看，其他参数都不行，那很可能是过拟合。

所以我们要测试：

```text

exit_z = 0.25, 0.5, 1.0
regression_window = 60, 90, 120

```

如果结果在一片参数区域里都还可以，说明策略逻辑更稳。

## 16. 容量分析怎么理解

题目要求策略能容纳 1000 万到 5000 万美元。

容量不是看回测收益，而是看：

```text
你的交易量会不会超过市场能承受的流动性？
```

简单估计方法：

```text
单日最大可交易金额 = ADV * 参与率
容量 = 单日最大可交易金额 / 平均交易权重
```

其中 ADV 是日均成交额，参与率可以假设 5%。

项目里如果没有真实成交量，就先用代理 ADV，并在报告里说明这是粗略容量估计。

对应代码：

```text
src/cross_market_mr/analysis.py

```

## 19. 这道题最容易被追问的问题

### 问题 1：为什么不是直接做 pairs trading？

因为 pairs trading 通常只处理两个资产之间的价差，而这里一个资产可能同时受 BTC、ETH、SPY、QQQ、SMH 影响。多因子残差模型更适合跨市场资产。

### 问题 2：为什么 beta 要滚动估计？

因为市场状态会变化。固定 beta 会把旧关系强行用到新市场，容易失效。滚动 beta 能适应关系变化，也避免全样本未来函数。

### 问题 3：如果残差不回归怎么办？

这就是策略风险。可以通过 ADF、止损、持仓期限、分散化、成本控制和参数敏感性分析来降低风险，但不能完全消除。

### 问题 4：策略 alpha 来自哪里？

来自跨市场定价不充分、投资者结构差异、加密 24/7 与美股交易时间错配、情绪定价和短期流动性冲击。

### 问题 5：为什么 Binance 跨市场环境重要？

因为它降低了跨市场交易摩擦。以前需要美股券商、加密交易所、多账户、多保证金、多转账，现在理论上可以在统一保证金体系下做多腿对冲。
